# Story 3.1: Generate PRD Artifact

Status: done

## Story

As an AI-native Engineer,
I want a machine-readable PRD generated from discovery state,
so that downstream architecture and task planning have stable requirement IDs.

## Acceptance Criteria

1. Given discovery state exists, when PRD generation runs, then `_mugiew-method/prd/PRD.md` is created with stable IDs for FRs, NFRs, ACs, Constraints, Architecture Decisions, Open Questions, and Assumptions.
2. Given assumptions exist, then assumptions are indexed.
3. Given requirements are generated, then requirements are written as capabilities, not implementation details.
4. Given PRD is generated, then it references source discovery artifacts.

## Dev Notes

- Governance engine owns generated PRD artifacts.
- Core owns project paths and safe writes.
- This story does not implement `/approve-prd`; it only generates the draft PRD artifact.
- Requirements should be capabilities. Avoid phrases like "use React", "create table", "install package" in Functional Requirements.

### API Guidance

- Prefer helper shape:

```ts
renderPrdArtifact(input)
writePrdArtifact({ projectRoot, input })
```

- [ASSUMPTION] Stable IDs can be generated by section prefix and input order for MVP: `FR-001`, `NFR-001`, `AC-001`, `C-001`, `AD-001`, `OQ-001`, `AS-001`.
